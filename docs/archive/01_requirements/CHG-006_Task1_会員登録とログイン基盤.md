# CHG-006 Task1: 会員登録とログイン基盤

## 背景・課題

会員機能の最初の障壁は「会員を識別する仕組み」が存在しないこと。
まずはバックエンドに会員アカウントと認証 API を追加し、以降 Step の土台を作る。

## ビジネス要件

### REQ-1-001: 会員登録
- メールアドレス・表示名・パスワードで会員登録できる
- メールアドレスは一意である
- 入力バリデーションを行う

### REQ-1-002: ログイン
- メールアドレス・パスワードでログインできる
- 認証成功時にトークンを発行する
- 認証失敗時は統一メッセージを返す

### REQ-1-003: ログアウト
- ログアウト API でトークンを無効化できる

### REQ-1-004: 会員情報取得
- ログイン中ユーザーが自分の会員情報を取得できる

## 受け入れ条件

### AC-1-001: 登録
- [ ] 新規会員を作成できる
- [ ] 既存メールアドレスでの登録はフロント上でエラー
- [ ] 不正入力は バリエーションで弾く

### AC-1-002: 認証
- [ ] 正しい資格情報でログインできる
- [ ] 不正な資格情報は エラー を返す
- [ ] ログアウト後、同じトークンは無効になる

### AC-1-003: 認証済み情報取得
- [ ] 認証済みでAPI が成功する
- [ ] 未認証でAPI は エラー を返す

## 対象外

- ゲストカート引き継ぎ
- 注文履歴
- 管理者ロール制御
