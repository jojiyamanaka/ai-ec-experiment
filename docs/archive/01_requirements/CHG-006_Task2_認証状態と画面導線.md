# CHG-006 Task2: 認証状態と画面導線

## 背景・課題

バックエンドに認証 API ができても、フロントエンドにログイン状態管理と画面導線がないと利用できない。

## ビジネス要件

### REQ-2-001: 認証状態管理
- フロントエンドでログイン状態を保持する
- リロード後もログイン状態を復元できる

### REQ-2-002: 認証 UI
- 会員登録画面とログイン画面を追加する
- ログアウト操作を提供する

### REQ-2-003: API 連携
- 認証が必要な API にトークンを付与する
- 認証エラー時はログイン導線へ遷移または再認証を促す

### REQ-2-004: 既存導線維持
- 未ログイン時でも既存の商品閲覧・カート・注文導線は維持する

## 受け入れ条件

### AC-2-001: ログイン状態
- [ ] ログイン後に画面上でログイン中ユーザーを識別できる
- [ ] ページ再読み込み後もログイン状態が復元される

### AC-2-002: 導線
- [ ] ヘッダー等からログイン/登録/ログアウトに遷移できる
- [ ] 認証エラー時にユーザーが復帰可能な導線がある

### AC-2-003: 互換性
- [ ] 未ログイン購入フローが壊れていない

## 対象外

- カート引き継ぎロジック
- 会員注文履歴
- 管理者ロール制御
