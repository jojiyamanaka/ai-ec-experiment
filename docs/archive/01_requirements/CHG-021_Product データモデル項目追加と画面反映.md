# CHG-021: Product データモデル項目追加と画面反映

作成日: 2026-02-20
---
## 1. 背景
- 現行の商品管理は最小構成であり、顧客画面と管理画面での運用に必要な情報が不足している。
- 商品分類の定義が十分に統一されておらず、商品参照と商品運用の一貫性を高める必要がある。
- 商品登録を含む運用を安定させるため、商品マスタ項目の拡張を要件として明確化する。

---
## 2. 要件
- 商品マスタの項目を拡張すること。
- 商品カテゴリをマスタとして管理できること。
- 商品はカテゴリを識別できる情報を保持すること。
- 管理画面で商品登録（新規作成）ができること。
- 管理画面で商品登録時に、拡張後の商品マスタ項目を設定できること。
- 顧客画面と管理画面で、同じ商品マスタ定義・カテゴリ定義を前提に商品を参照・運用できること。
- 商品情報として、品番・カテゴリ・価格・在庫・公開状態を一貫して扱えること。
- 商品画像の扱いは現行ルールを維持すること。
- 追加する商品項目が API 仕様（型）および画面入出力と整合していること。

---
## 3. 受け入れ条件
- 商品マスタ拡張項目が保持され、顧客画面と BO 画面で一貫して参照できること。
- 管理画面で新規商品を登録できること。
- 登録商品のカテゴリが顧客画面と BO 画面で一貫して表示されること。
- 商品情報（品番・カテゴリ・価格・在庫・公開状態）が一貫して保持・反映されること。
- 追加項目が API 仕様（型）と矛盾しないこと。

---
## 4. 仕様への反映
- 業務要件（`docs/requirements.md`）に、商品マスタ項目拡張・カテゴリ管理・商品登録要件を追記する。
- データモデル定義（`docs/data-model.md`）に、拡張後の商品マスタとカテゴリの関係を追記する。
- 商品仕様（`docs/specs/product.md`）に、商品マスタ拡張項目と商品登録項目の定義を追記する。
- BO 画面仕様（`docs/ui/admin-ui.md`）に、拡張項目を含む商品登録要件を反映する。
- 顧客画面仕様（`docs/ui/customer-ui.md`）に、拡張項目の表示要件を反映する。
- API 仕様（`docs/api/customer-bff-openapi.json` / `docs/api/backoffice-bff-openapi.json` / `docs/api/openapi.json`）に、追加項目の入出力を反映する。
