# CHG-022 impl-notes

## T-1
- `GET /api/bo-auth/me` は新規 service を増やさず、`BoAuthGuard` が確定した `request.boUser` をそのまま返す薄い controller 実装にした。
- 401 契約は既存の `BoAuthGuard` 例外整形（`BFF_UNAUTHORIZED` / `BFF_INVALID_TOKEN`）を再利用し、`me` エンドポイント側では独自エラーハンドリングを追加しない方針にした。
- ルート追加後の疎通確認は `login -> me(200) -> me without token(401)` を `backoffice-bff` コンテナ内 `curl` で実施し、設計書の契約条件を直接検証した。
